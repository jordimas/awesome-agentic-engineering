# Contribution Guidelines

Please note that this project is released with a [Contributor Code of Conduct](code-of-conduct.md). By participating in this project you agree to abide by its terms.

## Adding to the list

- Submit one pull request per resource.
- Add the resource to the bottom of the most relevant category, or propose a new category if needed.
- Use the following format: `- [Name](link) - Description.`
- Keep descriptions short, objective, and focused on what makes the resource useful for teams adopting agentic engineering.
- Start the description with a capital letter and end with a period.
- Check spelling and grammar.
- Make sure your text editor is set to remove trailing whitespace.
- The pull request title should be descriptive (e.g., "Add Resource Name").
- The pull request description should explain why the resource is worth adding to a curated list focused on practical, team-applicable agentic engineering.

## Quality bar

**This list aims to be a HIGH-LEVEL OVERVIEW of agentic engineering, NOT an exhaustive catalog.** We collect only SIGNIFICANT, high-impact resources that genuinely shape how teams work. Small, niche, or experimental entries clutter the map and defeat the purpose. When in doubt, leave it out.

- Resource must be actively maintained, accessible, and not behind a paywall.
- GitHub repositories must have more than 10,000 stars.
- Avoid duplicates and superseded tools.
- Prefer first-party documentation, well-known authors, or widely cited work.

## Updating an existing entry

Open a pull request that improves the description, fixes a broken link, or removes an entry that no longer meets the quality bar.

Thank you for your suggestions!
